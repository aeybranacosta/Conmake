# Conmake
🖋️ Project Overview
ConMake is a specialized digital workspace designed for the creation and documentation of constructed languages. Unlike traditional word processors that rely on pre-existing fonts, ConMake empowers linguists, world-builders, and hobbyists to hand-draw their own orthography from scratch.
The name ConMake stands for "Constructed Maker," reflecting the tool's goal: providing a raw, creative canvas where a language isn't just written, but visually engineered. From ancient stone-carved runes to fluid alien scripts, ConMake is the bridge between your imagination and a formalized writing system.
🚀 Core Features
1. The ConMake Initialization Engine
The workflow begins with the "+ Make Language" command. This initializes a unique session where you define the identity of your project. By naming your language within the ConMake environment, you set the foundation for a new linguistic database.
2. The 123-Slot Lexicon Framework
ConMake utilizes a rigid 123-slot architecture. This specific constraints-based design encourages creators to focus on the "Swadesh List" style of essential vocabulary.
Primary Slots (1-40): Ideal for fundamental concepts (Sun, Water, Life, Death).
Action Slots (41-80): Perfect for core verbs and directional markers.
Abstract Slots (81-123): Reserved for complex grammatical particles, tense markers, or numerical glyphs.
3. Integrated Glyph-Drawing Interface
Each slot in ConMake features a dedicated, high-response drawing canvas.
Vector-Style Tracking: The drawing engine tracks your mouse or stylus input in real-time, allowing for intricate strokes and detailed character design.
Independent Logic: Every one of the 123 slots functions as a standalone module. You can edit the symbol for Slot 15 without ever risking the data integrity of Slot 115.
One-Click Refinement: Each slot includes a "Clear" (C) function, allowing you to iterate on your character designs until the aesthetic of your language is perfectly consistent.
4. Direct Lexical Mapping
ConMake follows a strict (Word)=(Drawing) logic. By entering text into the word field, you create a permanent link between the meaning and the visual representation. This makes it an invaluable tool for maintaining consistency in your world-building projects.
🛠️ Technical Implementation
ConMake is built to be lightweight, portable, and easily hostable via GitHub Pages:
Architecture: Built entirely on the Vanilla Web Stack (HTML5, CSS3, and ES6 JavaScript).
Rendering: Uses the HTML5 Canvas API for zero-latency drawing.
Responsiveness: Features a CSS Grid layout that adapts to your screen size, ensuring you can "ConMake" on a desktop or a tablet.
Data Handling: Uses Base64 encoding for character saves, ensuring your drawings can be exported as high-quality image data.
📖 Operational Guide
Launch ConMake: Open the index.html file or visit your GitHub Pages link.
Define Your Language: Click + Make Language and type your chosen name (e.g., "ConMake-Runic").
Map Your Vocabulary: Scroll to a slot and type the "Source Word" in the text box.
Inscribe the Glyph: Use the canvas below the word to draw your custom character.
Save & Export: Use the Save button to finalize your entry. The system maps your drawing directly to the text provided.
Refine: Use the red C button if a character doesn't fit the visual "vibe" of your language.
🎨 Why Use ConMake?
For Writers: Visualizing your characters' handwriting or holy texts.
For DM/GMs: Creating handouts for players that they actually have to "decipher" using a key you made.
For Artists: Experimenting with logographic systems and abstract iconography.
